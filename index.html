Werewolves Mobile Built Summer 2014

This is my implementation of the party game werewolves/mafia. Its developed for the android and ios platforms using Apache Cordova with a Ruby-Sinatra backend on a Digital Ocean Server. I wanted to build this game for 3 purposes. 1) I love the game Mafia and was disappointed to not find a solid application on google play store that was cross platform capable. It difficult playing mobile games with your friends when they are divided between iOS and Android. 2) I had recently taught myself Sinatra and wanted to give myself a project so I could build a real API. 3) I had vision for a new genre of games dubbed 'TabletTop Gaming', essentially it would bring the joy of tabletop games like apples to apples, poker, munchkin and D&D to the 21 century. There have been mobile ports of these games but few have really captured the eseences of these games. Its the human element thats most exciting. Since everyone has a phone in their pocket now adays and almost every house hold has a tablet laying around the vision came to me. The tablet could be the gameboard while a person's phone is their hand. So I wanted to build a framework so you could easy introduce ur friends new games while we waited for dinner to arrive or just hanging out. At McHacks in Montreal this last winter I attempted to bring this vision to life using Bluetooth but I ran into a few issues with the Android Bluetooth API. I could connect with a most 7 other devices(a small number for my vision) and second you could only truly communicate with one pairing at a time. This made broadcast messaging impossible. In retrospect, I could have developed some sort of timed stacked package deliver system where the devices connect and reconnect every few seconds. Still I feel like that isn't a very stable system to build a prototype on. So rather than try and reinvent the wheel I've decided to forsake native applications for now and try to use a system that has already solved this issue --- Web Technology.

What I learned from this Project

This was my first attempt to build a cordova application with a backend. I had debated about how I wanted to do backend ultimately I went with Sinatra over Rails. While I had a solid years worth of experience using Rails, I recognized that it overly complicated what I wanted to do for the backend. I had built toy backends using rails (see my Android2RailsGCM) and wasn't the easiest experience so I decided to give Sinatra a try. It was the second most popular ruby framework and I had heard it was bare minimial. Which indeed it was, data migrations using Data-Mapper were difficult at first but eventually I figured it out. With Rails, there was a specific way to do things since Rails had been born in a vaccuum but with Sinatra it was really just pure ruby and incredibly mallible.

On the cordova client side of things, I had to come up with my own form of authenication because to get omniauth to work I would have use AJAX anyway so I went with a API key solution. One major issue I ran into was that JSONP isn't compatible with POSTs or any other HTTP request than GET. In my investigation it became clear that was because JSONP in order to work around the domain restrictions it embedded itself in a GET request. This behavior is kind of similiar to how Rails gets around some browsers not supporting PUT and DELETE. So I was forced to use JSON and remove ORIGIN restrictions on my Sinatra server. In the future, I might have to come up with a better authentication solution to protect against future attacks.
